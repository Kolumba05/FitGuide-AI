# FitGuide-AI
ТЗ проекта

## Общие сведения

Название проекта: Web-приложение подбора индивидуального питания и тренировок с использованием нейросети

Тип работы: Курсовая работа

Формат приложения: Web-приложение с возможностью установки как PWA и дальнейшей упаковки в мобильное приложение (WebView)

Срок завершения проекта: май 2026 года

Команда разработки: 3 человека

⸻

## Цель и назначение проекта

Целью проекта является разработка web-приложения, предназначенного для формирования персонализированных рекомендаций по питанию и физической активности на основе данных пользователя с использованием нейросетевых технологий.

Приложение ориентировано на пользователей, заинтересованных в контроле калорийности рациона, улучшении физической формы и отслеживании прогресса.

⸻

## Задачи проекта

Для достижения поставленной цели необходимо решить следующие задачи:
 • проанализировать предметную область персонализированного питания и тренировок;
 • спроектировать архитектуру web-приложения;
 • реализовать клиентскую и серверную части приложения;
 • разработать модуль расчёта калорийности и БЖУ;
 • интегрировать нейросетевой модуль для генерации рекомендаций;
 • обеспечить хранение пользовательских данных;
 • подготовить приложение к публикации;
 • оформить отчётную документацию по курсовой работе.

⸻

## Функциональные требования

- 4.1 Функции пользователя

Пользователь должен иметь возможность:
 • вводить персональные данные (возраст, рост, вес, пол);
 • выбирать цель (похудение, набор массы, поддержание формы);
 • указывать уровень физической активности;
 • получать расчёт суточной нормы калорий;
 • получать рекомендации по питанию;
 • получать тренировочный план;
 • просматривать историю изменений веса и рекомендаций.

⸻

- 4.2 Функции приложения

Приложение должно:
 • рассчитывать суточную норму калорий на основе формул базового обмена веществ;
 • рассчитывать распределение БЖУ;
 • сохранять данные пользователя;
 • передавать расчётные данные в нейросетевой модуль;
 • отображать результаты в удобном для пользователя виде.

⸻

- 4.3 Функции нейросети

Нейросеть должна:
 • анализировать входные параметры пользователя;
 • формировать персонализированные рекомендации по питанию;
 • генерировать пример меню на день или неделю;
 • формировать тренировочный план;
 • предоставлять текстовые пояснения выданных рекомендаций.

Результаты работы нейросети носят рекомендательный характер и не являются медицинскими назначениями.

⸻

## Нефункциональные требования
 • удобный и понятный пользовательский интерфейс;
 • корректная работа в современных браузерах;
 • модульная архитектура;
 • возможность масштабирования;
 • хранение данных в локальной или серверной базе данных;
 • защита пользовательских данных.

⸻

## Архитектура системы

Приложение должно состоять из следующих компонентов:
 • Frontend: web-интерфейс пользователя;
 • Backend: серверная часть, реализующая бизнес-логику;
 • База данных: хранение пользовательской информации;
 • Нейросетевой модуль: генерация персонализированных рекомендаций.

Взаимодействие компонентов осуществляется по REST API.

⸻

## Используемые технологии
 • Frontend: JavaScript (React)
 • Backend: Python, fastAPI
 • База данных: SQLite
 • Нейросеть: языковая модель, подключаемая через API
 • Среда разработки: Visual Studio Code
